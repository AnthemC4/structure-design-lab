# AI 热点速配 - 生产环境部署指南

## 方案 A：云服务器部署（推荐）

### 1. 服务器要求

| 配置 | 要求 |
|------|------|
| CPU | 1 核 + |
| 内存 | 1GB+ |
| 存储 | 10GB+ |
| 系统 | Ubuntu 20.04+ / CentOS 7+ |
| 网络 | 公网 IP |

### 2. 安装 Node.js

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS/RHEL
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs

# 验证安装
node -v
npm -v
```

### 3. 上传项目文件

```bash
# 创建目录
sudo mkdir -p /opt/hotspot
sudo chown $USER:$USER /opt/hotspot

# 上传文件（SCP 或 FTP）
# 将以下文件上传到 /opt/hotspot/
# - hotspot_server.js
# - ai_hotspot_auto.html
```

### 4. 安装依赖

```bash
cd /opt/hotspot
npm install express puppeteer node-cron nodemailer
```

### 5. 配置 Nginx

```bash
sudo apt-get install nginx  # Ubuntu
# 或
sudo yum install nginx  # CentOS
```

**Nginx 配置：**
```nginx
# /etc/nginx/sites-available/hotspot
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名或服务器 IP
    
    # 前端静态文件
    location / {
        root /opt/hotspot;
        index ai_hotspot_auto.html;
        try_files $uri $uri/ =404;
    }
    
    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**启用配置：**
```bash
sudo ln -s /etc/nginx/sites-available/hotspot /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 6. 启动后端服务

```bash
cd /opt/hotspot
nohup node hotspot_server.js > hotspot.log 2>&1 &

# 验证服务
curl http://localhost:3000/api/stats
```

### 7. 配置开机自启

```bash
# 创建 systemd 服务
sudo tee /etc/systemd/system/hotspot.service > /dev/null <<'EOF'
[Unit]
Description=AI Hotspot Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/hotspot
ExecStart=/usr/bin/node hotspot_server.js
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 启用服务
sudo systemctl daemon-reload
sudo systemctl enable hotspot
sudo systemctl start hotspot
sudo systemctl status hotspot
```

### 8. 配置防火墙

```bash
# Ubuntu UFW
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# CentOS firewalld
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### 9. 访问测试

浏览器访问：
```
http://your-domain.com
或
http://服务器 IP
```

## 方案 B：Netlify + 云服务器（混合部署）

### 前端部署到 Netlify

1. 访问 https://app.netlify.com/drop
2. 拖入 `ai_hotspot_auto.html`
3. 获得域名：`https://xxx.netlify.app`

### 后端部署到云服务器

1. 按方案 A 步骤 2-4 安装 Node.js 和上传后端
2. 启动服务：`node hotspot_server.js &`
3. 配置公网访问（开放 3000 端口）

### 修改前端 API 配置

编辑 `ai_hotspot_auto.html`：
```javascript
const API_BASE = 'http://服务器IP:3000';  // 替换为你的服务器 IP
```

重新部署到 Netlify。

## 方案 C：Docker 部署（最简单）

### 1. 创建 Dockerfile

```dockerfile
FROM node:18-alpine

WORKDIR /app
COPY hotspot_server.js .
COPY ai_hotspot_auto.html .

RUN npm install express puppeteer node-cron nodemailer

EXPOSE 3000

CMD ["node", "hotspot_server.js"]
```

### 2. 构建镜像

```bash
docker build -t hotspot-server .
```

### 3. 运行容器

```bash
docker run -d -p 3000:3000 --name hotspot hotspot-server
```

### 4. 访问

```
http://服务器 IP:3000
```

## 定时通知配置

### 钉钉通知

需要配置钉钉机器人 Webhook：

1. 钉钉群 → 群设置 → 智能群助手 → 添加机器人
2. 选择"自定义"机器人
3. 复制 Webhook 地址
4. 编辑 `hotspot_server.js` 中的 `sendDingtalkNotification` 函数

### 邮件通知

需要配置阿里云邮件推送：

1. 访问 https://dm.console.aliyun.com
2. 创建发信地址
3. 获取 SMTP 配置
4. 编辑 `hotspot_server.js` 中的 `sendEmailNotification` 函数

## 常见问题

### 1. 端口 3000 被占用

```bash
# 查看占用端口的进程
sudo lsof -i :3000

# 修改端口
# 编辑 hotspot_server.js，修改 PORT = 3001
```

### 2. Nginx 502 错误

```bash
# 检查后端服务
sudo systemctl status hotspot

# 重启后端
sudo systemctl restart hotspot
```

### 3. 热点数据不更新

```bash
# 检查日志
tail -f /opt/hotspot/hotspot.log

# 手动刷新
curl -X POST http://localhost:3000/api/hotspots/refresh
```

## 性能优化

### 1. 启用 HTTPS

```bash
# 安装 Certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com
```

### 2. 配置缓存

在 Nginx 配置中添加：
```nginx
location ~* \.(html|js|css|png|jpg)$ {
    expires 7d;
    add_header Cache-Control "public, immutable";
}
```

### 3. 启用 Gzip 压缩

```nginx
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript application/javascript application/xml+rss application/json;
```

## 安全建议

1. **配置防火墙**：只开放必要端口（80/443）
2. **定期更新**：保持系统和 Node.js 最新版本
3. **监控日志**：定期检查访问日志和错误日志
4. **备份数据**：定期备份热点数据和通知日志
5. **限制 API 访问**：生产环境建议配置 API 鉴权
